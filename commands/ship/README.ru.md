# /ship

Закоммитить изменения (стейджнутые или все) и запушить на remote за один шаг.

## Использование

```
/ship           — закоммитить стейджнутые изменения
/ship --yes     — закоммитить стейджнутые изменения, обойти защиту main/master
/ship all       — стейджнуть всё и закоммитить
/ship all --yes — стейджнуть всё и закоммитить, обойти защиту main/master
/ship dry       — показать стейджнутый diff и предлагаемое сообщение без коммита
```

## Поведение

- Генерирует сообщение коммита из diff'а, подстраивая стиль под недавние коммиты из `git log`
- Запрещает пуш напрямую в `main` или `master` без флага `--yes`
- В режиме `dry` выводит summary diff'а и предлагаемое сообщение, затем останавливается — ничего не коммитится и не пушится
- `--yes` обходит только проверку ветки; force-push не выполняется

## Заметки

- Чтобы просмотреть все изменения в режиме `dry`, сначала стейджни их: `git add -A`
- Если в репозитории ещё нет коммитов, используется стандартный imperative mood без подстройки под историю
- Сообщения коммитов — в повелительном наклонении ("Add feature", не "Added feature"), без упоминания AI-инструментов
